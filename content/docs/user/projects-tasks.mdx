Aika organizes your workspace using **Projects** and **Tasks**. Linking tasks to your time logs helps managers see exactly where time was spent.

---

## 1. Projects vs. Tasks

*   **Projects**: Long-term initiatives (e.g., `"Q3 Infrastructure Redesign"` or `"Product Launch"`). They have start and end dates.
*   **Tasks**: Actionable todo items (e.g., `"Design login page interface"`). A task can belong to one project, or exist independently.

---

## 2. Managing Tasks (Kanban vs. Backlog)

You can manage tasks from the **Task Board**:

```text
+-------------------+      +-------------------+      +-------------------+
|      TODO         |      |    IN PROGRESS    |      |      DONE         |
+-------------------+      +-------------------+      +-------------------+
| [ ] Fix header    | ===> | [/] Style login   | ===> | [x] Create schema |
| [ ] Add audit log |      |                   |      |                   |
+-------------------+      +-------------------+      +-------------------+
```

### Kanban Board
*   Drag and drop tasks between **Todo**, **In Progress**, and **Done** columns to update their status.
*   Click a card to edit description details, set priorities (`Low`, `Medium`, `High`), or set due dates.

### Backlog / List View
*   Tasks that aren't on the board sit in the **Backlog**.
*   This view is ideal for long lists of future tasks that aren't ready for active tracking.

---

## 3. Linking Tasks to Time Logs

Linking tasks is the key to granular reporting:

1.  When saving a time log (manual or after stopping a timer), click the **Link Tasks** field.
2.  Select one or more active tasks from the search dropdown.
3.  When you save the log, Aika creates a relationship:
    *   One task can have **multiple time logs** associated with it.
    *   One time log can contain **multiple tasks** if you worked on several small items at once.
